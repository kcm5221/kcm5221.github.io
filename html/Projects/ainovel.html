AI Novel

개발 기간 2025.07.04 ~ 2025.07.16

깃허브 링크

1. 프로젝트 개요

목표 : AI(LLM, Gemini CLI)와 바이브코딩 방식으로 협업하여 장편 소설 집필 워크플로우를 구현했습니다.
전반적인 코드는 AI가 작성하였으며, 저는 실행·테스트·워크플로우 설계와 데이터 구조 기획을 담당했습니다.
ChromaDB를 활용해 플롯 · 캐릭터 · 스타일 데이터를 벡터 형태로 관리하고,
“사람이 쓰는 원본 Markdown” ↔️ “AI 참고용 벡터DB” 이중 구조 로 효율적인 집필·검색·수정 과정을 실험했습니다.

2. 핵심 구현

  • AI 바이브코딩 기반 구현 – 전체 코드(novel_cli.py, split_aihub_story.py 등)는 AI가 작성, 저는 실행/디버깅 및 구조 설계를 진행
  • 소설 원본 ↔ 파생 데이터 분리 – novels/*.md 원본은 영구 보존, corpus_segments_v1 컬렉션은 스크립트로 재생성
  • Novel CLI – novel_cli.py 하나로 load · generate · review · update-plot · status 통합 관리
  • AIHub 코퍼스 이용 – split_aihub_story.py로 YAML 변환 후 Chroma 적재, 스타일 프롬프트에 자동 삽입
  • LLM 프롬프트 자동 생성 – 현재 플롯·캐릭터·이전 N 챕터 요약을 불러와 ‘다음 챕터’ 프롬프트 구성
  • 챕터 스냅샷 & 메타데이터 – 생성된 챕터는 파일 저장 + DB current_chapter 업데이트로 작업 재개 용이

3. 배운 점

- AI가 작성한 코드를 실행/검증하며 Python 스크립트 실행 흐름과 CLI 설계 방식을 이해했습니다.
- ChromaDB 벡터DB 개념, 데이터 적재/검색 과정, 프롬프트 구성 방식 등을 익혔습니다.
- AI 코드를 그대로 쓰는 것이 아니라 내 요구사항에 맞춰 프롬프트·구조를 조정하는 과정을 경험했습니다.
