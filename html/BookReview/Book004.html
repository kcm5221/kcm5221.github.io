그림과 작동 원리로 쉽게 이해하는 서버의 기초

2025.04.11



저자: 키하시 마사히로

발행일: 2023년

이 책은 서버가 무엇인지 모르는 사람도 서버의 기본 구조와 개념을 이해할 수 있도록 구성된 입문서다. 서버의 설치, 운영, 보안, 장애 대응까지 전반적인 흐름을 그림과 함께 설명해주고 있어, 서버에 대한 큰 틀을 잡기 적절한 책이다.

하지만 생소한 개념이 많아 이해에 많은 시간이 걸렸고, 여전히 명확하지 않은 부분도 있다. 그럼에도 서버라는 세계에 발을 들이는데 좋은 안내서였으며, 막연했던 개념들이 조금씩 구체화되었다.

1. 서버란?

  • 서버의 정의: 네트워크를 통해 접속한 클라이언트의 요청을 받아, 필요한 데이터를 제공하거나 기능을 수행해주는 컴퓨터 또는 프로그램
  • 역할: 웹 페이지를 보여주거나, 데이터베이스에 저장된 정보를 전달하고, 파일을 전송하거나 사용자 인증을 처리하는 등 다양한 기능을 수행한다.
  • 종류: 웹 서버(웹 페이지 제공), DB 서버(데이터 저장/관리), 애플리케이션 서버(비즈니스 로직 처리) 등 목적에 따라 역할이 분리되어 운영된다.

2. 서버 동작 원리

  • 서버의 동작 흐름: 서버는 클라이언트가 보내는 요청(Request)을 수신하고, 그 요청의 내용에 맞는 처리(예: 데이터 조회, 계산 등)를 수행한 뒤, 그 결과를 응답(Response)으로 돌려보낸다. 이 기본 구조는 웹 브라우저에서 페이지를 여는 행위부터 스마트폰 앱의 기능까지 대부분의 네트워크 서비스에 공통으로 적용된다.
  • 요청/응답 구조: 클라이언트(사용자)는 웹 브라우저나 앱을 통해 서버에 요청을 보내고, 서버는 그 요청에 따라 필요한 데이터를 찾아 응답한다. 이 과정은 TCP/IP 통신을 기반으로 작동하며, 요청과 응답은 모두 특정 형식(프로토콜)을 따라 이루어진다.
  • IP/포트/DNS: 서버는 인터넷상에서 고유한 IP 주소를 가지고 있으며, 하나의 서버에서 여러 서비스를 운영할 수 있도록 포트 번호를 사용한다. DNS는 사람이 기억하기 쉬운 도메인 이름을 IP로 변환하여 서버와 연결되도록 도와주는 시스템
  • 프로토콜: 클라이언트와 서버가 정보를 주고받기 위해 지켜야 하는 통신 규칙. 예를 들어 HTTP는 웹사이트를 불러올 때, FTP는 파일을 주고받을 때, SMTP는 이메일을 보낼 때 사용되며, 각 프로토콜은 정해진 형식에 따라 데이터를 전달한다.

3. 서버의 구조

  • 3계층 구조: 서버 시스템은 역할을 나누어 3단계로 구성된다. 클라이언트의 요청은 먼저 웹 서버(Web Server)에 도달해 외부 요청을 처리하고, 필요한 로직 처리는 앱 서버(Application Server)에서 수행한다. 이후 앱 서버는 데이터베이스 서버(DB Server)와 통신하여 정보를 조회하거나 저장한 뒤, 그 결과를 클라이언트에게 응답한다.
  • 가상 서버(Virtual Server): 하나의 물리적인 컴퓨터(서버) 안에 여러 개의 독립된 서버 환경을 소프트웨어적으로 구성하는 방식. 서버 자원을 여러 가상 머신으로 나눠 사용하며, 각각 독립적인 운영체제와 설정을 가질 수 있다. 이 방식은 물리 장비를 효율적으로 활용하고, 다양한 서비스를 유연하게 운영할 수 있게 해준다.

4. 서버의 구축과 운영

  • 운영체제 설치: 서버에는 일반적으로 리눅스 같은 안정적이고 보안성이 높은 운영체제가 설치됨.
  • 배포/설정: 서버가 어떤 서비스를 할지 정하고, 이를 실행할 소프트웨어를 설치 및 설정하며, 외부에서 접근할 수 있도록 포트를 개방하고 방화벽을 설정
  • 모니터링: 서버의 상태와 로그를 실시간으로 확인하며, 오류나 비정상적인 활동을 빠르게 탐지하여 대응할 수 있도록 함

5. 인터넷과 보안

  • 공통키/공개키 암호화: 데이터를 암호화하여 제3자가 볼 수 없게 만드는 기술로, 안전한 통신을 위해 사용
  • SSL/TLS: 서버와 클라이언트 간의 데이터를 암호화하여 중간에서 유출되지 않도록 하는 보안 프로토콜. HTTPS 주소로 시작되는 사이트에 적용된다.
  • VPN/CDN: VPN은 사용자의 위치를 숨기고 안전하게 통신할 수 있도록 돕고, CDN은 전 세계 여러 지역에 콘텐츠를 분산시켜 빠르게 전달하는 기술이다.

6. 장애 대책

  • RAID: 여러 개의 하드디스크를 조합하여 하나처럼 동작하게 해, 디스크 고장 시에도 데이터 손실을 방지하거나 복구가 가능하도록 한다.
  • UPS: 정전이 발생해도 일정 시간 동안 전력을 공급해 서버가 안전하게 종료되거나 계속 작동할 수 있게 한다.
  • 부하 분산: 여러 대의 서버에 작업을 나누어 처리하게 하여 하나의 서버에 부하가 집중되지 않게 한다.
  • 클러스터링: 같은 역할을 하는 서버를 여러 대 묶어 하나처럼 운용하여, 한 서버가 고장나도 다른 서버가 대신 작동하게 한다.
